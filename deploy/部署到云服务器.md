# 将项目更新到云服务器

## 方式一：用 Git（推荐）

### 1. 本机：提交并推送代码

```bash
# 在项目根目录
cd c:\work\AI漫剧工场\BigBanana-AI-Director-main

git add .
git commit -m "更新：API 改为 api.gitcc.com、depei 品牌、nginx 代理与 body 限制"
git push origin main
# 若主分支叫 master，则：git push origin master
```

### 2. 云服务器：拉取并重新构建

```bash
# SSH 登录到云服务器后
cd /path/to/BigBanana-AI-Director-main   # 你服务器上项目所在目录

git pull origin main   # 或 master

# 使用项目根目录的 docker-compose（端口 3005）
docker-compose down
docker-compose build --no-cache
docker-compose up -d

# 或使用 deploy 目录的 compose（端口 80）
# cd deploy
# docker-compose -f docker-compose.yaml down
# docker-compose -f docker-compose.yaml build --no-cache
# docker-compose -f docker-compose.yaml up -d
```

### 3. 首次在服务器上还没有项目时

```bash
git clone https://github.com/你的用户名/BigBanana-AI-Director.git
cd BigBanana-AI-Director
# 然后同上：docker-compose build --no-cache && docker-compose up -d
```

## 方式二：不用 Git，直接上传/同步

### 1. 本机打包（在项目根目录）

```bash
# 排除 node_modules、.git 等，只打包源码
tar --exclude=node_modules --exclude=.git --exclude=dist -czvf bigbanana-deploy.tar.gz .
# 或在 PowerShell 里用压缩工具打 zip，排除 node_modules、.git、dist
```

### 2. 上传到服务器

- 用 **scp**（在本机执行，把下面的 user、服务器IP、路径改成你的）：

```bash
scp bigbanana-deploy.tar.gz user@你的服务器IP:/home/user/
```

- 或用 **FTP / 宝塔 / 其它面板** 上传压缩包。

### 3. 服务器上解压并构建

```bash
ssh user@你的服务器IP

cd /home/user
mkdir -p BigBanana-AI-Director-main
tar -xzvf bigbanana-deploy.tar.gz -C BigBanana-AI-Director-main
cd BigBanana-AI-Director-main

docker-compose down
docker-compose build --no-cache
docker-compose up -d
```

## 端口说明

| 文件 | 端口映射 | 访问方式 |
|------|----------|----------|
| 项目根目录 `docker-compose.yaml` | 3005:80 | http://服务器IP:3005 |
| `deploy/docker-compose.yaml` | 80:80 | http://服务器IP（需 80 端口未被占用） |

## 常用命令

```bash
# 查看容器是否在跑
docker ps

# 查看日志
docker-compose logs -f

# 停止
docker-compose down

# 重启
docker-compose restart
```

---

## 云服务器上访问说明

- 部署完成后，在浏览器访问：**http://你的服务器IP:3005**（若用根目录 compose）。
- 数据存在浏览器 IndexedDB，换机器/换浏览器会看不到本机数据；API Key 也是浏览器里配置的。
- 若云服务器访问 api.gitcc.com 无 CORS 问题，则不需要走 `/api-proxy`；当前逻辑是「本地（localhost）才走代理」，在云服务器上会直连 api.gitcc.com，一般可正常使用。
